Для подключения абонентов по протоколу OpenVPN выполните следующие действия.

---

1. В меню **Защита > Сертификаты** создайте корневой и конечный [сертификаты](/index.php?article=78).

![Иллюстрация](../../../images/article_223/vpn_connection1.png)

При создании конечного сертификата на вкладке **«Использование ключа»** в поле **«Шаблон»** выберите значение **«VPN-сервер»**.

![Иллюстрация](../../../images/article_223/vpn_connection2.png)

Пример настроек конечного сертификата:

![Иллюстрация](../../../images/article_223/vpn_connection2-1.png)

2. В меню **Сеть > Провайдеры и сети** добавьте [OpenVPN-сеть](/index.php?article=198). Чтобы пользователи могли подключаться к ресурсам локальных сетей ИКС, на вкладке **«Основные настройки»** установите флаг **«Передать клиенту маршрут по умолчанию»** и выберите из списка сети, которые нужно маршрутизировать.

![Иллюстрация](../../../images/article_223/openvpn1-10.1.png)

3. На вкладке **«Шифрование и сертификаты»** укажите сертификаты, созданные в **Шаге 1**.

![Иллюстрация](../../../images/article_223/vpn_connection4.png)

4. В меню **Сеть > VPN > Пользователи** отметьте флагами пользователей, которым будет разрешено подключаться по протоколу OpenVPN. При этом система предложит выбрать, к какой из созданных сетей будет подключаться пользователь. Чтобы изменения вступили в силу, нажмите кнопку **«Сохранить»**.

![Иллюстрация](../../../images/article_223/two-factor_1.jpg)

5. Перейдите в [индивидуальный модуль](/index.php?article=142#tab7) пользователя на вкладке **«OpenVPN»**. Если необходимо настроить маршрутизацию из локальной сети ИКС до удалённых ресурсов в локальной сети клиента, укажите адреса этих сетей в поле «Удалённые сети за клиентом».

![Иллюстрация](../../../images/article_223/openvpn-ip.jpg)

6. В меню **Сеть > Провайдеры и сети** выберите созданную OpenVPN-сеть и выгрузите архив (`openvpn-configs.tar.gz`) с пользовательскими сертификатами (`*.ovpn`) для их подключения. Другой вариант: каждый пользователь должен самостоятельно зайти в свой личный кабинет и скачать свой сертификат в формате `*.ovpn`.

![Иллюстрация](../../../images/article_223/vpn_connection7.png)

7. Чтобы подключиться, пользователь должен установить на своем компьютере [утилиту OpenVPN](https://openvpn.net/client/) (OpenVPN), запустить ее и импортировать файл `*.ovpn`. После этого станет доступной возможность подключения.

Если используется другая утилита подключения к OpenVPN-серверу и появилось требование иметь необходимые файлы (`ca.crt`, `client.crt`, `client.key`, `ta.key`), создайте данные файлы вручную и [скопируйте](/index.php?article=221#files) в них информацию из файла `*.ovpn`.

> ⚠ Внимание! Клиенту нельзя назначать адреса ИКС, адреса других пользователей, адрес сети, широковещательный адрес сети и последний адрес в сети, который зарезервирован для DHCP.
>
> Множественное подключение пользователя к OpenVPN-сети невозможно. По одному конфигурационному файлу может быть только одно подключение к OpenVPN в один момент времени.

## Прекращение доступа пользователя к OpenVPN

Отозвать у пользователя доступ к OpenVPN можно в меню **Сеть > VPN > Пользователи**. Снимите флаг **«OpenVPN-доступ»** напротив нужного пользователя и нажмите **«Сохранить»**.

Доступ у пользователя прекратится сразу, даже если у него было активное соединение.

В меню **Защита > Сертификаты** созданный ранее автоматический сертификат пользователя станет серым, с состоянием «сертификат отозван».

![Иллюстрация](../../../images/article_223/vpn_connection8.png)

> ⚠ Внимание! Озванный сертификат не рекомендуется удалять из модуля «Сертификаты», иначе он пропадёт из списка отозванных и пользователь с таким сертификатом вновь получит доступ к OpenVPN.

При удалении из ИКС пользователя, который имеет доступ к OpenVPN, его сертификат также будет отозван.
