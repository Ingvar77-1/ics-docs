# Мониторинг

Модуль «Мониторинг» предназначен для просмотра статистики использования сетевых и системных ресурсов, а также различных показателей системы.

---

Модуль **«Мониторинг»** предназначен для просмотра статистики использования сетевых и системных ресурсов, а также различных показателей системы, таких как виртуальная память, загрузка процессора, загрузка системы, пинг до `ya.ru`, трафик на сетевых интерфейсах и других.

Для открытия модуля перейдите в меню **Обслуживание > Мониторинг**.

![Иллюстрация](../../images/article_114/monitoring1-10.2.jpg)

В модуле расположены следующие вкладки:

- Графики
- Мониторинг состояния системы
- Настройки службы Аудит
- Настройки Zabbix-агента
- Журнал

## Графики

В левой части данной вкладки отображается список доступных показателей. На каждый из выбранных пунктов в правой части вкладки строится несколько графиков, различных по временному интервалу: за последний час, 6 часов, день, неделю и т. д.

![Графики](../../images/article_114/monitoring1-11.png)

При необходимости можно добавить собственные графики для проверки доступности указанных хостов. Для этого нажмите кнопку **«Добавить»** и укажите название и адрес.

![Добавление графика](../../images/article_114/monitoring1-2-10.2.jpg)

При нажатии на кнопку **«Очистить»** все графики системы будут удалены и сформированы заново.

## Мониторинг состояния системы

На данной вкладке отображаются следующие сведения о службе:

- статус службы (запущен, остановлен, выключен, не настроен);
- кнопка **«Включить»** (**«Выключить»**) — позволяет запустить или остановить службу;
- журнал последних событий.

![Мониторинг состояния системы](../../images/article_114/monitoring2-10.2.jpg)

## Настройки службы Аудит

Служба Аудит предназначена для отслеживания потока событий в режиме реального времени.

![Служба Аудит](../../images/article_114/monitoring3-10.2.jpg)

Вкладка предназначена для настройки службы аудита.

1. Выберите, какие **события** служба должна просматривать:
   - все события;
   - выбрать вручную (установить нужное значение отдельно для каждого пункта).

> ⚠ Важно! Не рекомендуется включать аудит в режиме «Все события > Отслеживать все события», так как ИКС будет уходить в циклическую перезагрузку.

2. Нажмите **«Сохранить»**.

## Настройки Zabbix-агента

Zabbix — это система, которая позволяет собирать различные данные с устройств. С её помощью можно производить мониторинг сети, серверов, виртуальных машин, баз данных, приложений и устройств. Начиная с версии 10.2 в ИКС добавлен Zabbix-агент, при этом Zabbix-сервер необходимо развернуть самостоятельно, где вам удобно.

![Zabbix-агент](../../images/article_114/monitoring4-10.2.jpg)

Для настройки Zabbix-агента выполните следующие действия:

1. Выберите один из **режимов работы**:
   - **Пассивный**. Укажите параметры:
     - адреса для прослушивания — это адреса, на которых будет слушать Zabbix-агент. По умолчанию (если оставить поле пустым) он слушает на всех интерфейсах;
     - порт прослушивания — порт, на котором будет слушать Zabbix-агент. По умолчанию 10050;
     - Zabbix-серверы — серверы Zabbix, с которыми будет связываться агент. Здесь можно указать IP-адреса и доменные имена (1.1.1.1, test.ru).
   - **Активный**. Укажите параметры:
     - активные сервера — сервера, для получения активных проверок. Здесь можно указать IP-адреса, доменные имена и IP-адреса с портами (1.1.1.1, test.ru, 1.1.1.1:10051);
     - частота обновлений списка активных проверок (в секундах). По умолчанию 120 сек.
2. При необходимости установите флаг **«Автоматически создавать разрешающее правило»**, чтобы пропускать трафик на порт прослушивания (для локальных сетей).
3. Если требуется, измените **уровень отладки** (логов). По умолчанию установлен уровень «Предупреждения».
4. Нажмите **«Сохранить»**.

## Журнал

На данной вкладке отображается сводка всех системных сообщений модуля с указанием даты и времени.

![Журнал](../../images/article_114/monitoring5-10.2.jpg)

Журнал является стандартным элементом веб-интерфейса ИКС.
