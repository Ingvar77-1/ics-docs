# Настройка узлов кластера

---

Для удобства настройки лучше открыть рядом две вкладки с веб-интерфейсом ИКС.

После настройки и запуска SLAVE-ноды доступ к веб-интерфейсу по ее начальному адресу (например 172.16.1.160:81) прекратится. Нужно настраивать VM, которая выступает в качестве ПК локальной сети SLAVE-ноды, и заходить по адресу управления, (например, 192.168.1.2:81).

Используйте возможности создания snapshots в Proxmox, если у вас есть такая возможность или подобный гипервизор.

В PVE машины должны иметь соответствующие VLAN (tag) на нужных сетевых интерфейсах.

## ИКС-А (MASTER)

Настройка включает в себя следующие шаги:

1. Установите и зарегистрируйте ИКС как Демо.
2. Настройте WAN: em0 172.16.1.161 + web/ssh доступ.
3. Настройте LAN: em1 192.168.1.254/24 + web/ssh доступ.
4. Добавьте администратору IP-адрес 192.168.1.13.
5. Добавьте пользователей и настройте сервисы (начальные для проверки).
6. Рекомендуется делать снапшот*, если у вас размещение на гипервизоре.
7. Перейдите в меню «Обслуживание > Кластер > Настройки».
8. IP кластера — 10.0.0.1/30. Интерфейс — re0.
9. Порт обмена статусами узлов — 10101. Можно задать свой, если он свободен (есть проверка).
10. Порт синхронизации данных — 22022. Можно задать свой, если он свободен (есть проверка).
11. IP-адрес доступа к веб-интерфейсу — 192.168.1.1/24.
12. Интерфейс управления — em1 SSH-ключ этой ноды (сгенерирован). Скопируйте в поле SSH-ключ парной ноды на ИКС-В.
13. Пока пропустите мониторинг локальных сетей.
14. IP-адрес парной ноды — 10.0.0.2.
15. SSH-ключ парной ноды. Возьмите из поля SSH-ключ этой ноды (сгенерирован) ИКС-В.
16. Допустимая загрузка системы за 15 минут — по умолчанию равна количеству CPU*3. При достижении загрузкой системы этого значения произойдет переключение узлов кластера.
17. Включите службу, просмотрите журнал.
18. Дождитесь перехода в состояние RISES->MASTER (Основная).

## ИКС-В (SLAVE)

Настройка включает в себя следующие шаги:

1. Установите и зарегистрируйте ИКС как Демо.
2. Добавьте администратору IP-адрес 192.168.1.13.
3. Рекомендуется делать снапшот*.
4. Перейдите в меню «Обслуживание > Кластер > Настройки».
5. IP кластера — 10.0.0.2/30. Интерфейс — re0.
6. Порт обмена статусами узлов — 10101. Должен быть как у MASTER.
7. Порт синхронизации данных — 22022. Должен быть как у MASTER.
8. IP-адрес доступа к веб-интерфейсу — 192.168.1.2/24.
9. Интерфейс управления — em1.
10. SSH-ключ этой ноды (сгенерирован). Скопируйте в поле SSH-ключ парной ноды на ИКС-А.
11. Пока пропустите мониторинг локальных сетей.
12. IP-адрес парной ноды — 10.0.0.1.
13. SSH-ключ парной ноды. Возьмите из поля SSH-ключ этой ноды (сгенерирован) ИКС-А.
14. Допустимая загрузка системы за 15 минут — по умолчанию равна количеству CPU*3. Оставить по умолчанию.
15. Включите службу, просмотрите журнал.

## Сеть

Таблица с примером минимального количества настроек нод для включения кластера.

| ИКС | Меню | А | В |
|---|---|---|---|
| cluster state |  | MASTER | SLAVE |
| em0 | Сети | 172.16.1.161 | выключен (DOWN) |
| em1 | Сети | 192.168.1.254 | выключен (DOWN) |
| em1 | Кластер: IP-адрес доступа к веб-интерфейсу | 192.168.1.1/24 | 192.168.1.2/24 |
| ro0 | Кластер: IP-адрес узла | 10.0.0.1/30 | 10.0.0.2/30 |

> ⚠ Внимание! Адрес ноды всегда записывается на интерфейс локальной сети (em1), чтобы по нему всегда можно было получить доступ к GUI ИКС.

> ⚠ Внимание! Если в ИКС создано перенаправление для 22 порта, необходимо выбрать конкретный интерфейс, иначе резервный узел не пройдет аутентификацию на основном, кластер не будет работать.
