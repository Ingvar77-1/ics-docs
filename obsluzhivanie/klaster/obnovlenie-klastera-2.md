# Обновление кластера

Обновление узлов кластера имеет свои особенности. Рекомендуем ознакомиться с особенностями перед процедурой обновления кластера.

> ⚠ Важно! Когда выключается Резервный узел кластера, на нем удаляется текущая база данных ИКС.

## Порядок обновления узлов

| Шаг | Основной | Резервный |
|-----|----------|-----------|
| 1 |  | Выключить службу кластер до сообщения в журнале: `Cluster service correct termination complete successful. Bye` |
| 2 | Выключить службу кластер до сообщения в журнале: `Cluster service correct termination complete successful. Bye` |  |
| 3 | Сделать резервную копию настроек |  |
| 4 | Выгрузить резервную копию настроек на внешний ресурс или носитель |  |
| 5 | Скачать и установить обновления штатно |  |
| 6 | Проверить работу сети и сервисов для пользователей |  |
| 7 | Если обновление ломает работу, выполнить процедуру отката версии. При необходимости развернуть резервную копию из шага 4 |  |
| 7 | Если обновление прошло успешно, сделать новые настройки службы кластер (SSH-ключи) | Настроить провайдера. Скачать и установить обновления штатно. Сделать новые настройки службы кластер (SSH-ключи) |
| 8 | Включить службу кластер. Дождаться получения статуса **Основной** |  |
| 9 |  | Включить службу кластер. Дождаться получения статуса *Резервный* |
